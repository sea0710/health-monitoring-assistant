# 愈见·血常规监测助手 - 设计上下文

## Design Context

### Users
**主要用户**：化疗患者及其家属

**使用场景**：
- 化疗后居家监测血常规指标
- 复查前查看指标趋势
- 接收复查提醒通知
- 查看分级参考标准

**使用频率**：按需偶尔使用（非日常高频应用）

**用户心理**：处于治疗期或康复期的患者及家属，需要简单易用的工具来追踪健康数据，期望获得专业、可靠的信息支持。

### Brand Personality
**品牌性格**：温暖、可靠、简洁

**情感目标**：
- **安心**：通过清晰的信息呈现和专业的数据解读，让用户感到踏实
- **信任**：通过严谨的设计和准确的信息，建立用户对产品的信任
- **掌控感**：通过简洁的界面和流畅的交互，让用户轻松管理健康数据

**品牌调性**：医疗健康类应用，需要在专业性和亲和力之间找到平衡。不应过于冷冰冰的医疗感，也不应过于随意轻浮。

### Aesthetic Direction
**视觉风格**：
- **极简克制**：大量留白，信息精简，避免视觉噪音
- **专业严谨**：清晰的信息层级，数据导向的设计

**设计原则**：
1. **内容优先**：界面服务于内容，减少装饰性元素
2. **层级清晰**：通过字号、字重、颜色建立明确的信息层级
3. **留白呼吸**：合理运用空间，让界面有呼吸感
4. **一致性**：统一的视觉语言，降低用户认知负担
5. **可读性**：确保文字清晰易读，数据一目了然

**主题模式**：仅浅色模式

**色彩系统**：
- **主色调**：青绿色 #14b8a6（温暖而专业的医疗色调）
- **辅助色**：基于主色的渐变色系
- **功能色**：成功（绿色）、警告（橙色）、危险（红色）
- **中性色**：灰色系用于文字、边框、背景

### Design Principles
基于用户需求和品牌定位，制定以下设计原则：

1. **温暖而不失专业**
   - 使用圆润的卡片和按钮，传达温暖感
   - 保持清晰的数据呈现，体现专业性
   - 避免过于冰冷的技术感设计

2. **简洁而不简陋**
   - 精简信息，只展示必要内容
   - 通过留白和层级创造高级感
   - 每个元素都有存在的意义

3. **清晰而不拥挤**
   - 建立明确的视觉层级（标题→正文→辅助信息）
   - 使用充足的间距，避免信息过载
   - 重要信息突出显示，次要信息适度弱化

4. **一致而不单调**
   - 统一的设计语言贯穿所有页面
   - 相同功能使用相同的视觉表达
   - 在一致性的基础上保持适当的视觉变化

5. **易用而不复杂**
   - 简化操作流程，减少用户思考
   - 清晰的视觉反馈，让用户知道发生了什么
   - 符合用户直觉的交互设计

### Anti-References
**应避免的设计**：
- 过于花哨的渐变和特效
- 复杂的动画和过渡效果
- 信息密度过高的界面
- 缺乏层级的平铺式设计
- 过于医疗化、冷冰冰的界面
- 过于随意、缺乏专业感的界面
