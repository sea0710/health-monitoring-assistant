# 体验版小程序版本管理策略

## 一、现状分析

### 1.1 项目信息
- **小程序AppID**: wx62b4bee890679ebe
- **当前版本**: 2.1.12 (package.json)
- **发布状态**: 仅体验版，不正式发布
- **有效期**: 体验版7天有效期

### 1.2 挑战
- **版本有效期短**: 体验版每7天需要重新生成二维码
- **用户访问**: 用户需要重新扫码进入体验版
- **版本管理**: 缺乏版本控制和发布记录
- **回滚能力**: 出现问题时难以快速回滚到稳定版本

## 二、版本管理策略

### 2.1 版本号规范

#### 版本号格式
```
主版本号.次版本号.修订号-体验版标识
```

**示例**:
- `2.1.12-beta1` - 第1个体验版
- `2.1.12-beta2` - 第2个体验版
- `2.1.13-beta1` - 次版本更新后的第1个体验版

#### 版本号规则
- **主版本号**: 重大功能变更
- **次版本号**: 新功能添加
- **修订号**:  bug修复和小改动
- **体验版标识**: 每次更新体验版时递增

### 2.2 版本管理流程

#### 1. 版本规划
- 在 `package.json` 中更新版本号
- 创建版本发布说明文档
- 记录本次更新的功能和修复

#### 2. 构建与测试
- 执行 `npm run build` 构建项目
- 进行功能测试和兼容性测试
- 确保核心功能正常运行

#### 3. 发布体验版
- 在微信开发者工具中上传代码
- 生成体验版二维码
- 记录发布时间和版本信息

#### 4. 版本监控
- 收集用户反馈
- 监控关键功能的使用情况
- 记录版本使用数据

#### 5. 版本更新
- 根据反馈和测试结果进行优化
- 递增体验版标识或版本号
- 重新发布体验版

### 2.3 版本控制工具

#### 版本记录文件
创建 `VERSION.md` 文件记录版本历史：

```markdown
# 版本历史

## 2.1.12-beta1 (2026-04-02)
- 优化病历夹页面加载速度
- 添加加载动画和骨架屏
- 修复报告加载慢的问题

## 2.1.11-beta1 (2026-03-25)
- 更新白细胞参考范围
- 修复登录页面bug
- 优化OCR识别
```

#### 环境变量配置
在 `.env` 文件中添加版本信息：

```env
# 版本信息
APP_VERSION=2.1.12-beta1
BUILD_DATE=2026-04-02
```

### 2.4 体验版管理

#### 二维码管理
- 每次发布新体验版时生成新的二维码
- 将二维码保存在 `qrcodes/` 目录
- 命名格式: `version-betaX-date.png`

#### 有效期管理
- 设置体验版过期提醒
- 提前3天准备新版本
- 确保版本衔接不中断

#### 用户通知
- 在小程序内添加版本更新提示
- 通过微信消息通知用户
- 提供新版本的功能介绍

## 三、技术实现

### 3.1 版本信息展示

在小程序首页添加版本信息：

```javascript
// app.js
App({
  globalData: {
    version: '2.1.12-beta1',
    buildDate: '2026-04-02'
  }
})

// home.js
Page({
  onLoad() {
    const app = getApp()
    this.setData({
      version: app.globalData.version
    })
  }
})
```

```wxml
<!-- home.wxml -->
<view class="version-info">
  <text class="version-text">版本: {{version}}</text>
</view>
```

### 3.2 版本检查机制

添加版本检查功能，提示用户更新：

```javascript
// utils/version.js
const checkVersion = async () => {
  try {
    const remoteVersion = await fetchRemoteVersion()
    const localVersion = getApp().globalData.version
    
    if (compareVersions(remoteVersion, localVersion) > 0) {
      wx.showModal({
        title: '版本更新',
        content: '发现新版本，是否更新？',
        success: (res) => {
          if (res.confirm) {
            wx.setClipboardData({
              data: 'https://体验版二维码链接',
              success: () => {
                wx.showToast({ title: '二维码链接已复制，请在微信中打开' })
              }
            })
          }
        }
      })
    }
  } catch (error) {
    console.error('版本检查失败:', error)
  }
}
```

### 3.3 版本回滚方案

#### 代码分支管理
- 维护 `main` 分支作为稳定版本
- 创建 `develop` 分支进行开发
- 每个版本创建对应的标签

#### 回滚流程
1. 从版本标签 checkout 代码
2. 重新构建和测试
3. 发布为新的体验版
4. 通知用户使用新版本
